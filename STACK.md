| Категория       | Технология                                      |

| Язык            | TypeScript                                      |
| Стили           | Tailwind CSS                                    |
| UI-компоненты   | Radix UI + class-variance-authority (shadcn)    |

## База данных

| Среда     | Технология                          |
|-----------|-------------------------------------|
| Prod      | PostgreSQL (локальный)              |
| Dev/Test  | H2 (in-memory)                      |

### Схема таблиц

- `users` — пользователи (роли: SUPERUSER, USER)
- `trails` — маршруты (название, локация, сложность, координаты)
- `events` — события (название на ru/en/de: `title`, `title_ru`, `title_en`, `title_de`; дата, маршрут, организатор, координаты, статус)
- `event_participants` — участники событий
- `audit_log` — лог действий

## Auth — Supabase

| Компонент      | Описание                                                    |
|----------------|-------------------------------------------------------------|
| Auth           | Google OAuth + Email/Password                               |
| Поток токена   | Supabase token → `POST /api/auth/supabase` → Spring JWT     |
| Lazy auth      | Гость видит события на месяц (без деталей), вход по клику  |

Supabase используется **только для аутентификации**. Все данные приложения хранятся в PostgreSQL через Spring.

## Роли пользователей

| Роль        | Описание                                                                 |
|-------------|--------------------------------------------------------------------------|
| SUPERUSER   | Владелец, разработчики, администраторы приложения                        |
| USER        | Любой зарегистрированный пользователь. Может смотреть ленту, нажимать «Пойду», получать уведомления |
| Организатор | Не глобальная роль — пользователь, создавший событие, становится его организатором (`organizer_id` в таблице `events`) |
